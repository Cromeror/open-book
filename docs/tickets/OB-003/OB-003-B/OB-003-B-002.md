## Descripcion

Implementar endpoints REST para el CRUD de apartamentos, anidados bajo la copropiedad correspondiente.

## Contexto Tecnico

### Archivos afectados
- `src/api/routes/apartments.routes.ts`
- `src/api/controllers/apartments.controller.ts`

### Enfoque tecnico
Endpoints anidados bajo copropiedad. Solo admins de la copropiedad pueden gestionar.

## Criterios de Aceptacion

- [ ] POST /api/condominiums/:condoId/apartments
- [ ] GET /api/condominiums/:condoId/apartments
- [ ] GET /api/condominiums/:condoId/apartments/:id
- [ ] PATCH /api/condominiums/:condoId/apartments/:id
- [ ] DELETE /api/condominiums/:condoId/apartments/:id
- [ ] Filtros: por torre, por estado, por piso
- [ ] Solo admins de la copropiedad tienen acceso

## Notas de Implementacion

Crear apartamento:
```
POST /api/condominiums/:condoId/apartments
Body: {
  "number": "101",
  "floor": 1,
  "towerId": "uuid-tower" // opcional
}
```

Listar con filtros:
```
GET /api/condominiums/:condoId/apartments?tower=uuid&floor=1&active=true
```

Response:
```json
{
  "data": [
    {
      "id": "uuid",
      "numero": "101",
      "piso": 1,
      "torre": { "id": "uuid", "nombre": "Torre A" },
      "isActive": true
    }
  ],
  "total": 50,
  "page": 1,
  "limit": 20
}
```
