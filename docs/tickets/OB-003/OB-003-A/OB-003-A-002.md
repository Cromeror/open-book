## Descripcion

Implementar los endpoints REST para crear, leer, actualizar y desactivar copropiedades. Solo usuarios con rol ADMIN pueden realizar estas operaciones.

## Contexto Tecnico

### Archivos afectados
- `src/api/routes/condominiums.routes.ts`
- `src/api/controllers/condominiums.controller.ts`
- `src/services/condominium.service.ts`
- `src/api/validators/condominium.validator.ts`

### Enfoque tecnico
REST API con arquitectura en capas. Proteger endpoints con middleware de autenticacion y autorizacion.

## Criterios de Aceptacion

- [ ] POST /api/condominiums - Crear copropiedad
- [ ] GET /api/condominiums - Listar copropiedades
- [ ] GET /api/condominiums/:id - Obtener copropiedad
- [ ] PATCH /api/condominiums/:id - Actualizar copropiedad
- [ ] DELETE /api/condominiums/:id - Desactivar (soft delete)
- [ ] Todos los endpoints requieren rol ADMIN
- [ ] Validacion de datos de entrada

## Notas de Implementacion

Endpoints:
```
POST   /api/condominiums
GET    /api/condominiums?page=1&limit=10
GET    /api/condominiums/:id
PATCH  /api/condominiums/:id
DELETE /api/condominiums/:id
```

Response crear (201):
```json
{
  "id": "uuid",
  "nombre": "Conjunto Residencial Los Pinos",

  "cantidadUnidades": 120,
  "isActive": true,
  "createdAt": "2025-01-01T00:00:00Z"
}
```
