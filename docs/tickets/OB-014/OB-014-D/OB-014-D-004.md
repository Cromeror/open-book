## Descripcion

Crear la estructura de archivos para el modulo de Aportes Reales con validacion de permisos configurada. Este scaffold sera el punto de partida para implementar OB-007.

**IMPORTANTE**: Los aportes son inmutables - nunca se eliminan, solo se anulan con justificacion.

## Estructura de Archivos

```
apps/web/src/app/(dashboard)/contributions/
├── page.tsx                    # Lista de aportes
├── register/
│   └── page.tsx                # Registrar nuevo aporte
├── [id]/
│   ├── page.tsx                # Detalle del aporte
│   └── void/
│       └── page.tsx            # Anular aporte (con justificacion)
└── _components/
    └── .gitkeep
```

## Permisos por Pagina

| Pagina | Guard | Permiso |
|--------|-------|---------|
| /contributions | requireModule | contributions |
| /contributions/register | requirePermission | contributions:create |
| /contributions/[id] | requirePermission | contributions:read |
| /contributions/[id]/void | requirePermission | contributions:manage |

## Criterios de Aceptacion

- [ ] Estructura de carpetas creada
- [ ] Cada pagina verifica permisos correctamente
- [ ] Placeholder indica epic de implementacion (OB-007)
- [ ] Placeholder de anulacion indica que requiere justificacion
- [ ] Build pasa sin errores

## Referencia de Implementacion

La logica de negocio se implementara en **OB-007: Aportes Reales**:
- Registro con: apartamento, actividad, monto, fecha, medio de pago
- Estados: pendiente, confirmado, anulado
- Comprobantes adjuntos
- Trazabilidad total (quien, cuando, por que)
- **INMUTABILIDAD**: Anular, nunca eliminar
