## Descripcion

Crear la estructura de archivos para el modulo de Objetivos de Recaudo con validacion de permisos configurada. Este scaffold sera el punto de partida para implementar OB-004.

## Estructura de Archivos

```
apps/web/src/app/(dashboard)/goals/
├── page.tsx                    # Lista de objetivos
├── new/
│   └── page.tsx                # Crear objetivo
├── [id]/
│   ├── page.tsx                # Detalle del objetivo
│   ├── edit/
│   │   └── page.tsx            # Editar objetivo
│   └── states/
│       └── page.tsx            # Gestionar ciclo de vida
└── _components/
    └── .gitkeep                # Placeholder para componentes locales
```

## Permisos por Pagina

| Pagina | Guard | Permiso |
|--------|-------|---------|
| /goals | requireModule | goals |
| /goals/new | requirePermission | goals:create |
| /goals/[id] | requirePermission | goals:read |
| /goals/[id]/edit | requirePermission | goals:update |
| /goals/[id]/states | requirePermission | goals:manage |

## Criterios de Aceptacion

- [x] Estructura de carpetas creada
- [x] Cada pagina verifica permisos correctamente
- [x] Placeholder indica epic de implementacion (OB-004)

## Contenido del Placeholder

Cada pagina debe mostrar:
- Titulo de la pagina
- Descripcion breve
- Caja con:
  - Nombre del modulo
  - Permiso requerido
  - Epic de implementacion (OB-004)

## Referencia de Implementacion

La logica de negocio se implementara en **OB-004: Objetivos de Recaudo**:
- CRUD completo de objetivos
- Estados: borrador, activo, pausado, finalizado, cancelado
- Vinculacion con actividades
- Calculo de progreso
