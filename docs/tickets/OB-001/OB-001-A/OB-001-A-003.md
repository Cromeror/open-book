## Descripcion

Configurar herramientas de linting y formateo de codigo para mantener consistencia y calidad en todo el proyecto. ESLint para analisis estatico y Prettier para formateo automatico, integrados con la configuracion del monorepo Nx.

## Contexto Tecnico

### Archivos afectados
- `.eslintrc.json` o `eslint.config.js` (raiz del monorepo)
- `.prettierrc`
- `.prettierignore`
- `package.json` (scripts y dependencias)
- `apps/api/.eslintrc.json` (configuracion NestJS)
- `apps/web/.eslintrc.json` (configuracion Next.js)

### Enfoque tecnico
Usar ESLint con configuracion para TypeScript (@typescript-eslint) y Prettier integrado. Nx provee configuracion base que extiende para NestJS y Next.js.

## Criterios de Aceptacion

- [x] ESLint configurado con soporte TypeScript
- [x] Prettier configurado con reglas consistentes
- [x] Integracion ESLint-Prettier sin conflictos
- [x] Scripts pnpm: lint, lint:fix, format
- [ ] Pre-commit hook con husky (opcional pero recomendado)
- [x] Configuracion extendida para apps/api (NestJS) y apps/web (Next.js)

## Notas de Implementacion

- Nx incluye configuracion ESLint base que se puede extender
- Usar flat config de ESLint (eslint.config.js) si es ESLint 9+
- Configurar reglas especificas para async/await y promesas
- Incluir plugin para imports ordenados
- Considerar lint-staged para commits mas rapidos

Scripts en package.json:
```json
{
  "scripts": {
    "lint": "nx run-many --target=lint",
    "lint:fix": "nx run-many --target=lint --fix",
    "format": "nx format:write",
    "format:check": "nx format:check"
  }
}
```
