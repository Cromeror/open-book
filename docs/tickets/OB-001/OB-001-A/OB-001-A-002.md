## Descripcion

Configurar TypeScript para el proyecto con opciones estrictas que garanticen la calidad del codigo y faciliten el desarrollo. Incluir configuracion de path aliases para imports mas limpios.

## Contexto Tecnico

### Archivos afectados
- `tsconfig.json`
- `tsconfig.build.json` (opcional, para produccion)
- `package.json` (dependencias de desarrollo)

### Enfoque tecnico
Instalar typescript como dependencia de desarrollo y crear tsconfig.json con modo estricto habilitado, target ES2022, y path aliases configurados.

## Criterios de Aceptacion

- [ ] TypeScript 5.x instalado como devDependency
- [ ] tsconfig.json con strict: true
- [ ] Path aliases configurados (@/api, @/services, @/models, etc.)
- [ ] Output directory configurado como dist/
- [ ] Soporte para decoradores habilitado (para ORMs)
- [ ] Archivos de declaracion (.d.ts) generados

## Notas de Implementacion

- Usar target: "ES2022" para soporte de features modernas
- Configurar moduleResolution: "bundler" o "node16"
- Incluir resolveJsonModule: true para importar JSON
- Configurar sourceMap: true para debugging

## Testing

- [ ] tsc compila sin errores
- [ ] Los path aliases funcionan correctamente
- [ ] El directorio dist/ se genera correctamente
