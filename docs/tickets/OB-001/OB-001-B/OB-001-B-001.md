## Descripcion

Configurar la conexion a PostgreSQL, incluyendo la creacion de la base de datos de desarrollo y la configuracion de la cadena de conexion. PostgreSQL es elegido por su robustez, soporte ACID y capacidades de auditoria.

## Contexto Tecnico

### Archivos afectados
- `.env` (DATABASE_URL)
- `.env.example` (ejemplo de conexion)
- `docker-compose.yml` (opcional, para desarrollo local)

### Enfoque tecnico
Usar PostgreSQL 15+ con cadena de conexion configurada via variable de entorno. Opcionalmente proveer docker-compose para facilitar el desarrollo local.

## Criterios de Aceptacion

- [x] Variable DATABASE_URL definida en .env.example
- [x] Conexion verificable a PostgreSQL
- [x] docker-compose.yml con servicio postgres
- [x] Documentacion de como crear la base de datos (docs/DATABASE.md)

## Notas de Implementacion

- Formato de DATABASE_URL: `postgresql://user:password@host:port/database`
- Considerar base de datos separada para tests
- Configurar pool de conexiones apropiado
- Usar SSL en produccion

## Testing

- [x] La aplicacion puede conectarse a la base de datos
- [x] docker-compose up levanta PostgreSQL correctamente
- [x] La conexion falla con mensaje claro si la BD no esta disponible
