## Descripcion

Instalar y configurar Vitest como framework de testing del proyecto. Vitest ofrece velocidad superior y compatibilidad nativa con TypeScript, ideal para el stack del proyecto.

## Contexto Tecnico

### Archivos afectados
- `vitest.config.ts`
- `package.json` (scripts y dependencias)
- `tests/` (directorio de tests)
- `tests/setup.ts` (configuracion global)

### Enfoque tecnico
Instalar vitest y configurar con soporte para TypeScript, path aliases, y globals. Crear estructura de carpetas para tests unitarios e integracion.

## Criterios de Aceptacion

- [x] Vitest instalado y configurado
- [x] Path aliases funcionando en tests
- [x] Scripts: test, test:watch, test:coverage
- [x] Archivo setup.ts para configuracion global
- [x] Test de ejemplo ejecutando correctamente

## Notas de Implementacion

- Configurar globals: true para describe/it/expect sin imports
- Usar coverage con v8 o istanbul
- Configurar timeout apropiado para tests de integracion
- Separar configuracion de tests unitarios e integracion

## Testing

- [ ] pnpm test ejecuta tests correctamente
- [ ] pnpm test:coverage genera reporte
