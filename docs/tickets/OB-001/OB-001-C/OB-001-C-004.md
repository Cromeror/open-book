## Descripcion

Configurar GitHub Actions para ejecutar automaticamente linting, type-checking y tests en cada pull request y push a main. Esto garantiza la calidad del codigo antes de merge.

## Contexto Tecnico

### Archivos afectados
- `.github/workflows/ci.yml`
- `.github/workflows/pr-checks.yml` (opcional)

### Enfoque tecnico
Crear workflow de GitHub Actions que ejecute lint, typecheck, y tests con PostgreSQL como servicio. Configurar cache de dependencias para optimizar tiempos.

## Criterios de Aceptacion

- [ ] Workflow CI ejecuta en push a main y PRs
- [ ] Lint check (ESLint) ejecuta correctamente
- [ ] Type check (tsc --noEmit) ejecuta correctamente
- [ ] Tests ejecutan con PostgreSQL service
- [ ] Cache de node_modules configurado
- [ ] Status checks requeridos para merge

## Notas de Implementacion

- Usar actions/setup-node@v4
- Configurar PostgreSQL service para tests
- Pasos: checkout, setup node, install, lint, typecheck, test
- Configurar timeout razonable (10-15 min)
- Considerar matrix para multiples versiones de Node (opcional)

## Testing

- [ ] El workflow ejecuta exitosamente en un PR de prueba
- [ ] Falla apropiadamente si hay errores de lint
- [ ] Falla apropiadamente si hay errores de TypeScript
- [ ] Falla apropiadamente si hay tests fallidos
