## Descripcion

## Notas de Implementacion

Estados y transiciones:
```
ACTIVO
  -> PAUSADO (requiere justificacion)
  -> FINALIZADO (cuando se alcanza objetivo o decide cerrar)
  -> CANCELADO (requiere justificacion)

PAUSADO
  -> ACTIVO (reanudar)
  -> CANCELADO (requiere justificacion)

FINALIZADO
  -> (terminal, no hay transiciones)

CANCELADO
  -> (terminal, no hay transiciones)
```

```typescript
// Transiciones validas entre estados de objetivo
const TRANSITIONS: Record<ObjectiveStatus, ObjectiveStatus[]> = {
  ACTIVE: ['PAUSED', 'COMPLETED', 'CANCELLED'],
  PAUSED: ['ACTIVE', 'CANCELLED'],
  COMPLETED: [],
  CANCELLED: []
};

interface TransitionConfig {
  requiresJustification: boolean;
  validations?: ((objective: FundraisingObjective) => Promise<void>)[];
}

function canTransition(from: ObjectiveStatus, to: ObjectiveStatus): boolean {
  return TRANSITIONS[from].includes(to);
}
```
