## Descripcion

Implementar el hashing seguro de contrasenas usando bcrypt antes de almacenarlas en la base de datos. Las contrasenas NUNCA deben almacenarse en texto plano.

## Contexto Tecnico

### Archivos afectados
- `src/utils/password.ts`
- `src/services/auth.service.ts`
- `package.json` (dependencia bcrypt)

### Enfoque tecnico
Crear utilidades para hashear y comparar contrasenas usando bcrypt con un numero apropiado de salt rounds.

## Criterios de Aceptacion

- [ ] bcrypt instalado como dependencia
- [ ] Funcion hashPassword(password) implementada
- [ ] Funcion comparePassword(password, hash) implementada
- [ ] Salt rounds configurables via variable de entorno
- [ ] Las contrasenas se hashean antes de guardar en BD

## Notas de Implementacion

```typescript
import bcrypt from 'bcrypt';

const SALT_ROUNDS = parseInt(process.env.BCRYPT_SALT_ROUNDS || '12');

export async function hashPassword(password: string): Promise<string> {
  return bcrypt.hash(password, SALT_ROUNDS);
}

export async function comparePassword(
  password: string,
  hash: string
): Promise<boolean> {
  return bcrypt.compare(password, hash);
}
```

- Usar minimo 10-12 salt rounds
- bcrypt es async, usar await
- Considerar argon2 como alternativa mas moderna

## Testing

- [ ] hashPassword genera hash diferente al password original
- [ ] El mismo password genera hashes diferentes (por salt)
- [ ] comparePassword retorna true para match correcto
- [ ] comparePassword retorna false para match incorrecto
